# Deployment Guide: Daycare Finder Android

This guide outlines the process for building and deploying the Daycare Finder Android application.

## Build Process

### Development Build
For internal testing and development, use the Expo development server:
```bash
npm start
```
Open the app using the Expo Go app on a physical device or an Android emulator.

### Production Build (APK/AAB)
To generate a production-ready build, use the Easy Build (EAS) system:

1. **Install EAS CLI**:
   ```bash
   npm install -g eas-cli
   ```
2. **Configure EAS**:
   ```bash
   eas build:configure
   ```
3. **Run Android Production Build**:
   ```bash
   eas build --platform android
   ```
   This will generate an `.aab` (Android App Bundle) for Google Play Store upload or an `.apk` for direct installation.

## Deployment Checklist

- [ ] **Environment Variables**: Ensure all production API keys and endpoints are configured in the EAS build secrets.
- [ ] **App Icons & Splash**: Verify that `app.json` points to the correct production assets.
- [ ] **Permissions**: Ensure `app.json` contains all necessary Android permissions (Location, Camera, etc.).
- [ ] **Test Suite**: Ensure all tests pass in the CI/CD pipeline before triggering a production build.
- [ ] **Version Bump**: Increment the version code and version name in `app.json`.

## Distribution

### Google Play Store
1. Log in to the Google Play Console.
2. Create a new release and upload the `.aab` file generated by EAS.
3. Fill in the store listing details and submit for review.

### Internal Distribution
For beta testing, distribute the `.apk` file via Firebase App Distribution or directly to testers.
